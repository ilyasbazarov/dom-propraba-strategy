# dom-propraba-strategy · Стратегия и плейбуки Dom Prorab

База знаний стратегического контура Dom Prorab. Знание живёт в плоском наборе
md-файлов; модели-агенты в одноразовых чатах читают доки по SHA-pinned raw-URL,
выполняют свой слой работы и возвращают session-блок; владелец коммитит.
Канон метода — `_METHOD.md`. Проектные правила — `S05_CONVENTIONS.md`
(при расхождении S00/S05 приоритетнее канона).

---

## Как запустить сессию (3 шага)

1. Открой свежий чат на модели нужной роли (таблица ниже).
2. Первым сообщением вставь файл роли (или его текст) + строку
   `SHA: <актуальный commit-SHA>` + вопрос/задачу сессии.
3. В конце сессии забери session-блок и/или готовые файлы → закоммить →
   пришли новый SHA следующей сессии.

## Маршрутизация: какую роль звать

| Ситуация | Роль | Модель | Вход |
|---|---|---|---|
| Стратегический вопрос: L0/L1, канонизация STABLE-доков, портфель ставок, метод/оргструктура, «посоветоваться по прогрессу», эскалация от архитектора | **CEO** | Fable | `_CEO.md` + SHA |
| Планирование задач L2–L4, ревью открытых вопросов, приёмка выхода сессии, споры внутри исполнения, discovery-классификация | **Архитектор** | Opus | `_ARCHITECT.md` + SHA |
| Сгенерить бриф следующей задачи, уже расписанной в S04 | **Генератор** | Sonnet (механика) / Opus (нужны решения) | `_GENERATOR.md` + SHA |
| Исполнить задачу по готовому брифу | **Исполнитель** | Sonnet | `briefs/BRIEF-<ID>.md` + SHA |
| Применить session-блок к файлам, скомпоновать/запатчить доки | **Applier** | Haiku | `_APPLIER.md` + блок + SHA |
| T-INT-N · интервью фактов | Архитектор + владелец | Opus | `_ARCHITECT.md` + S10 (OPEN) |
| T-REV-N · ежемесячное ревью портфеля | **CEO** (данные готовит архитектор) | Fable | `_CEO.md` + S02/S04/S07 |
| T-RED-N · red-team STABLE-дока | Свежий адверсарный чат | Fable/Opus | бриф по T6 (S08) |

Иерархия суждения: **Владелец → CEO → Архитектор → Исполнитель → Applier.**
Финальное слово всегда за владельцем.

## Правило доступа (жёстко)

Агенты ходят в репо ТОЛЬКО через curl по commit-SHA, который прикладывает
владелец. Не web-fetch (кэширует), не ветка main «по умолчанию»,
не api.github.com (лимитируется без авторизации).

    curl -s https://raw.githubusercontent.com/ilyasbazarov/dom-propraba-strategy/<SHA>/<FILE>.md
