# S00_CHARTER · Хартия проекта «Стратегия и плейбуки Dom Prorab»

**Стабильность:** STABLE — изменения только через ADR (S06) с пройденной T-RED.
**Версия:** v1.0 · 2026-07-06

---

## 1. Что это за проект

База знаний стратегического контура Dom Prorab. Проект существует как md-файлы
в этом репо. Модели-агенты в одноразовых чатах читают доки по SHA-pinned raw-URL,
выполняют свой слой работы, возвращают session-блок; человек коммитит.
Метод — по S05_CONVENTIONS.

**Цель проекта:** упаковать стратегическое сырьё в целостную картину
L0→L4 и превратить её в исполняемые плейбуки для команды.

## 2. Бизнес-контекст

**Dom Prorab** — DIY-гипермаркет (ОсОО «Гамма тулс»), Бишкек, Кыргызстан.
Пост-санация, bootstrap-капитал, работа на собственные деньги.
2 розничных магазина (Бишкек/Кокжар + Беловодск), запускаемый e-commerce
dompro.kg (1P), программа Каталог 2.0.
Владелец (Ilnar) — единственный носитель стратегии и исполнитель-технарь.
Кадровый потенциал команды низкий: плейбуки пишутся так, чтобы исполнитель
без стратегического мышления мог по ним действовать.

Финансовая база: помесячный кэшфлоу и cash-basis P&L
(`inbox/raw_03`, `inbox/raw_05`) — калибратор проектных аппетитов.

## 3. L0 VISION (черновик, дотачивается в T-S2)

> Превратить Дом Прораба в кэш-машину через рост оборачиваемости и захват
> e-com-ниши до прихода сильных игроков. Стать топ-оф-майнд по DIY
> для ритейла и прорабов. Дать рынку e-com «как в Москве»: клиент выбирает
> с дивана и получает доставку день-в-день/на следующий день в дверь,
> включая крупногабарит — вместо базарного опыта (см. inbox/raw_04).

Статус: DRAFT. Канонизация — ADR после T-S2 + T-RED.

## 4. Финансовые красные линии

1. **Финансовый предохранитель.** Ни одна ставка не стартует без проверки
   «потянет ли казна и когда». Расходы можно размазывать по месяцам.
   Цель: движение к vision без банкротства по дороге.
2. **Runway floor.** Казна не опускается ниже N месяцев OPEX.
   - Значение N: **OPEN** (S10/Q-01). Выводится аналитически после получения
     остатка казны от владельца.
   - OPEX-база (черновик): core OPEX за последние 3 мес ≈ 1.38М сом/мес
     (см. raw_03 §2); определение OPEX финализируется ADR.
   - **Правило-заглушка до закрытия Q-01: денежный апрув любой ставки
     ЗАБЛОКИРОВАН.** Аналитическая и проектировочная работа не блокируется.
3. **WIP-лимит ставок.** Одновременно активных L2-ставок — не более 2–3.
   Точное число: **OPEN** (S10/Q-02), выводится аналитически из
   человекоресурса + ёмкости казны. До закрытия действует консервативный
   дефолт: **не более 1 активной ставки**.

## 5. Рамка слоёв (жёсткий контракт)

    L0  VISION        — куда идём (одно направление, на которое работает всё)
    L1  ВЕКТОРЫ       — стратегические направления / бизнес-модели
    L2  СТАВКИ (беты) — инициативы внутри вектора, на которые тратим ресурсы
    L3  ПРОЕКТЫ       — конкретные исполняемые единицы внутри ставки
    L4  ЗАДАЧИ        — атомарные действия с ID и приёмкой

Правила:
- Элемент нижнего слоя не существует без родителя в верхнем.
- Ни один документ и ни одна сессия не смешивает слои.
- Спор о принадлежности к слою решается ADR.
- Изменение L0–L1 — только через ADR с пройденной T-RED.
- Стандарты качества L1-вектора и карточки L2-ставки — в S08_TEMPLATES.

## 6. Scope

**В скоупе:** стратегия Dom Prorab (розница, e-com, B2B, Каталог 2.0,
импорт, партнёрские комплитеры), плейбуки для команды, финансовая рамка ставок.

**Вне скоупа:** операционка GCP-пайплайна dom-prorab-catalog (отдельный проект),
проект lombard-ops, проект holika, бизнес по розливу масла (Кара-Балта) —
кроме случаев, когда они пересекаются со стратегией как активы/ресурсы.

## 7. Уровни стабильности доков

| Уровень | Правило изменения |
|---|---|
| STABLE | Только через ADR + T-RED (S00, S08; L0/L1 в S02 после канонизации) |
| SEMI-STABLE | ADR или явное решение владельца (S01, S03, S05, S09) |
| LIVING | Правится рабочими сессиями через session-блоки (S02, S04, S07, S10; S06 — append-only) |
