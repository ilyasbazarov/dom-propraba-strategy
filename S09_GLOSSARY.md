# S09_GLOSSARY · Термины простым русским

**Стабильность:** SEMI-STABLE. Правило: термин, использованный в плейбуке
или S-доке и отсутствующий здесь, — дефект документа.
**Обновлено:** 2026-07-06 (сессия S0)

---

## Метод проекта

| Термин | Простыми словами |
|---|---|
| ADR | Запись о принятом решении: что решили, почему, можно ли откатить. Живут в S06, не переписываются |
| FACT | Цифра или факт, подтверждённые владельцем или его учётом. Живёт в S01 с источником и датой |
| ASSUMPTION | Допущение: похоже на правду, но не проверено. Живёт в S07, ждёт проверки. Использовать в денежных решениях нельзя |
| CONTEXT GAP | Стоп-сигнал модели: «мне не хватает вот этого, без него дальше нельзя». Вместо выдумывания |
| Session-блок | Обязательный итог каждой рабочей сессии: что сделано, что менять в доках, какие решения, какие вопросы владельцу |
| One-way door | Необратимое решение (или дорогое в откате). Требует полного ADR и паузы на обдумывание |
| Two-way door | Обратимое решение. Решается быстро, фиксируется одной строкой |
| Red-team (T-RED) | Сессия-«разнос»: отдельный чат, задача — найти дыры в документе и доказать, что план не сработает |
| T-INT | Интервью с владельцем: закрытие накопленных вопросов, ответы превращаются в FACT'ы |
| T-REV | Ежемесячная проверка каждой активной ставки: продолжаем / меняем курс / убиваем |

## Стратегическая рамка

| Термин | Простыми словами |
|---|---|
| L0 / Vision | Куда идём. Одно направление, на которое работает всё остальное |
| Вектор (L1) | Крупное направление бизнеса (например: e-com, реэкспорт). Обязан объяснять, какую слабость рынка использует |
| Ставка (L2) | Инициатива внутри вектора, на которую тратим деньги и время. Оформляется карточкой до выделения денег |
| Диагноз | Ответ на вопрос «какую проблему рынка мы используем». Вектор без диагноза — хотелка |
| PR-FAQ | Пресс-релиз о запуске, написанный ДО старта, + неудобные вопросы с честными ответами. Проверка, что идея продумана |
| Pre-mortem | Упражнение «прошёл год, ставка сдохла — почему?». Пишется до старта |
| Tripwire | Заранее установленный порог: «если к дате X метрика ниже Y — делаем Z». Решение принято в спокойном состоянии |
| Kill-критерий | Условие, при котором ставку убиваем, не жалея вложенного |
| WIP-лимит | Максимум одновременно активных ставок. Защита от распыления денег и людей |
| Riskiest Assumption First | Сначала проверяем то допущение, ложность которого убивает ставку целиком, — самым дешёвым способом |

## Финансы

| Термин | Простыми словами |
|---|---|
| Cash basis | Учёт по движению денег: доход — когда деньги пришли, расход — когда ушли. Без начислений и запасов |
| COGS | Стоимость проданного товара: оплаты поставщикам + импорт/таможня |
| OPEX | Регулярные расходы на работу бизнеса: зарплаты, аренда, банк, подписки. Без закупа товара и разовых вложений |
| CAPEX | Разовые вложения: оборудование, ремонт, доработки систем |
| EBITDA | Прибыль до налога и разовых статей: выручка − COGS − OPEX |
| Валовая маржа | (Выручка − COGS) / выручка. Сколько остаётся с каждого сома продаж до расходов на работу бизнеса |
| GMROI | Сколько валовой прибыли в год приносит каждый сом, замороженный в товаре. 0.63 = плохо, деньги стоят; 1.2+ = цель |
| Runway floor | Неприкосновенный минимум казны: N месяцев OPEX. Ниже — нельзя, что бы ни происходило |
| Казна | Все деньги компании: счета + наличные по всем точкам |

## Ассортимент и рынок

| Термин | Простыми словами |
|---|---|
| Локомотив | Товар, который тащит трафик и оборот: высокий спрос, стабильные продажи |
| Комплитер | Товар «для полноты полки»: сам мало зарабатывает, но без него каталог неполный |
| 1P | Продаём свой товар со своего склада |
| 3P | На нашей витрине продаётся товар партнёра с его склада |
| SKU | Одна товарная позиция в каталоге |
| KVI | Товары, по ценам которых покупатель судит «дорогой магазин или нет» |
